## Executive Summary

The user wants ONE app combining:
- **Notion's** flexible databases + block editor
- **ClickUp's** task management + views
- **ElevenReader's** AI voice reader + document TTS

**Target:** ADHD devs who vibe code, have too many ideas, need to capture fast, read faster, and stay focused.

**Market Gap:** NO app does all three. No app is ADHD-optimized at this level. This is a $0 competition field.

## PART 2: THE ADHD USER PROBLEM

### What ADHD Devs Actually Need

From competitor research + ADHD app analysis:

1. **Voice-first capture** — Thinking faster than typing. Need to dump brain without friction.
2. **Minimal UI** — No clutter. No overwhelm. No 47 buttons staring back.
3. **3-item daily focus** — Pick 3 tasks max. Don't let ADHD paralysis happen.
4. **Document TTS** — Can't read. Can listen. Need realistic voices.
5. **Offline-first** — Can't rely on connectivity. Ideas happen anywhere.
6. **Local data** — Don't trust cloud-only. Own your brain dumps.
7. **No punishment** — Streaks that punish aren't ADHD-friendly. Gentle nudges.
8. **AI that helps** — Task extraction, summarization, but NOT AI bloat everywhere.

## PART 3: v0.2 FEATURE ROADMAP

### Phase 1: Foundation (Core Experience)
**Goal:** Get voice capture + AI task extraction + realistic TTS working.

#### 1.1 Realistic AI Voices (FREE)
**Priority:** CRITICAL
**Why:** Web Speech API sounds robotic. Users hate it.

**Winner: Kokoro TTS**
- MOS score: 4.2 (beats ElevenLabs in blind tests!)
- 100% FREE, Apache 2.0 license
- Runs on CPU, Raspberry Pi, your server
- 54 preset voices, no voice cloning needed
- 210x real-time on GPU, 5x on CPU

**Backup: Chatterbox** (if voice cloning needed)
- MIT license, free
- Matches ElevenLabs in blind tests (63.75% preference!)
- Clone from 5-10 seconds audio

**Decision:** Use Kokoro (primary) + Chatterbox (optional cloning)
**Reason:** Both free, both better-than-or-equal-to ElevenLabs, both fully offline

#### 1.2 Voice Capture Fix
**Priority:** CRITICAL
**Why:** Mic not working in current build.
**Fixes needed:**
- Use Web Speech API for capture (works in mobile Chrome/Safari)
- Add permission handling with clear prompts
- Implement visual feedback (waveform, recording indicator)
- Add pause/resume capability

#### 1.3 AI Task Extraction
**Priority:** HIGH
**Why:** Core value prop. Voice → structured tasks.
**Implementation:**
- OpenAI GPT-4o mini or Claude Haiku for extraction
- Local fallback: rules-based extraction for offline
- Stream response for perceived speed

#### 1.4 Custom Pronunciation Dictionary
**Priority:** HIGH
**Why:** Tech terms (npm, GitHub, API) mispronounced.
**Implementation:**
- User-editable word → pronunciation map
- Auto-detect from context
- Sync across devices
